痛点发掘：
 
“我们的智能生活一直在进步，但我们的祖父母却还停留在原地”
中国有2.8亿老人，潮汕三市就有227.4万60岁以上人口，其中很多人被数字化抛弃。
 
和很多老人沟通后的调研结论：
“不知道如何在手机app上预约挂号，不知道如何进行叫车，不知道应该如何拨打视频电话“
 我们可以很轻松的总结出：现在市面上的智能设备ui交互太复杂了
但他们是真的不会么？
 
他们是怕：
害怕按错
害怕误扣费
听不懂简短的提示（需要手把手带着操作）
不会用命令格式下达指令
对普通话和充满机械感的语音比较陌生，难以建立信任感
 
我们通过母语交互和大模型意图识别能力来降低潮汕地区老人对智能设备的使用门槛，舒缓由此产生的恐惧心理，让老人更好的融入智能生活。
引用《“十四五”国家老龄事业规划》核心要义：“加快适老产品的智能交互技术研发，绝不是简单的字体放大。

技术壁垒：
-插入架构图 -
·  方言 ASR 容错与纠偏机制： 针对潮汕话特有的吞音和非标准化表达，我们在大模型前置了“特定领域 Prompt 纠错层”，识别准确率比裸调开源模型提升 X%。
·  模糊自然语言 -> 结构化 API 的意图路由（Intent Routing）： 老人说“太吵了”，系统不搜索视频，而是映射为 Volume_Down API。我们构建了专门的适老意图知识库。
·  极简边缘流式交互： 抛弃传统 App 层层递进的点击逻辑，采用 Web 端单点直达，降低端侧设备算力要求。

方式方法：
作为第三方技术供应商，集成在机顶盒中；我们直接提供现成且包含本地服务接口的成品

1. 基于已经订阅的服务，单独增加9.9/月的长辈关怀价值包，和运营商按照5:5分成 2. 直接收取技术授权费用

1.服务订阅：5049万-3102.5万=1946.5万/年 利润率38%

2. 养老地产/康养中心
床位测算逻辑：
广东全省养老机构共1726家，社区养老服务机构和设施超过2.1万个。养老床位45.25万张，其中，养老机构床位24.73万张（护理型床位占59.9%），社区养老服务机构和设施床位20.52万张；据测算，潮汕三市养老床位约在5万张左右。
方式方法：
帮助养老机构将普通的按铃呼叫改为意图识别智能响应，有效降低护工的巡房成本，提高整体监护效率
商业模式：
软硬件一体化改造费用+服务订阅费用
收入测算：首先从高端/公立示范性床位（已有620张——670张，到2026年年底预期示范床位可增加1000张），每张床位的改造费+首年订阅费约500元/年，1600位*500（首年改造费+订阅费）=80万
服务订阅费：300元/年，第二年服务订阅费1600位*300年=48万
成本测算：
•硬件改造成本：150（硬件采购价）*1600=24万
•Token消耗：1次交互按照消耗1000token来测算，大约消耗0.02元人民币，假设每床位平均交互10次，成本大致在0.2元人民币；0.2*1600*365=116800元 约为12万人民币
￮逻辑同上，控制token成本
利润测算：
•第一年：80-24-12=44万
•第二年：48-12=36万，利润率75%
